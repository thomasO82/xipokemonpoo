Deseign patern : Facon d'organiser son code. 
MVC : model view controller 

Model : Classe de description d'un objet. chaque table de la base de donné doit avoir son modele. (sauf bien sur les tables intermediare)
View  : bloc de code qui gere l'affichage en d'autre terme le html
Controller : Class qui gere la logique de notre application. 

Dans le dossier src, on met dossier models et controllers.

Dans le dossier view on y met le php qui gere l'affichae (html)

dans le dossier "public" on y met le index.php le SEUL fichier qui sera servi dans notre application

Un repository est une classe qui sera etendu par le modele, qui contiens TOUTES les requette a la base de donnée
